# 서브 에이전트 작업 가이드 및 규칙 (agent_rules.md)

이 문서는 모두의 방(ModuRoom) 개발 과정에서 에이전트들이 준수해야 할 공통 규칙과 한국어 가이드를 규정합니다.

## 1. 다국어 및 한국어 가이드 (RULE[user_global] 준수)

- **수정 및 보강 내용 번역:** 모든 개발 작업 결과, 코드 구조 변경 및 검증 내용은 사용자에게 항상 한국어로 상세히 번역 및 정리하여 제공해야 합니다.
- **UI 및 스타일 변경 금지:** UI 마크업 구조나 CSS 스타일 요소는 절대로 건드리지 마십시오. 기존 템플릿과 디자인 표준(Navy, Light Blue, Point Orange 및 폰트 크기/간격)은 유지하며, 비즈니스 및 통신 로직만 수정 및 구현해야 합니다.

## 2. 코드 스타일 및 아키텍처

- **ES6+ JavaScript:** 최신 JavaScript 문법을 따르며, 모듈화(ES Modules 혹은 CommonJS에 따라 프로젝트 설정 준수)를 명확히 해야 합니다.
- **예외 처리:** 비동기 API 통신, OCR 파싱, 이미지 Exif 제거 모듈에서 발생할 수 있는 모든 에외 상황에 대응하는 에러 캐칭을 필수로 탑재합니다.
- **EXIF 데이터 제거:** 임차인의 프라이버시 보호를 위해 이미지 파일에서 EXIF 메타데이터를 바이너리 수준에서 파싱하여 확실하게 삭제해야 합니다.

## 3. 성능 기준

- **화면 지연 시간:** 로컬 캐싱 및 비동기 처리 최적화를 통해 임대인/임차인 대시보드 스위칭 시 지연 시간 0.3초 이내를 유지합니다.
- **OCR 속도:** Mock OCR 및 API 로딩 시간이 3초 이내에 사용자 뷰로 반환될 수 있도록 타임아웃 및 시뮬레이션을 최적화합니다.
